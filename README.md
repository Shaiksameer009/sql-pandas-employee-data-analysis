# sql-pandas-employee-data-analysis
Employee database analysis using SQLite, SQL queries, and Pandas for data processing and comparison of SQL vs Pandas techniques.

# Project Overview
This project demonstrates how to create and analyze a synthetic employee database using SQLite, SQL queries, and Pandas. The objective is to understand how SQL and Pandas can be used together for data storage, querying, and analysis.

# Technologies Used
- Python
- SQLite
- Pandas
- Google Colab

# Dataset
A synthetic employee dataset with 40 records was generated using Python. Each employee record contains:

- emp_id
- name
- department
- salary
- years_experience
- performance_score

Departments include:
Engineering, Sales, Marketing, HR, and Finance.

# Tasks Performed

# Task 1: Database Creation
- Created SQLite database `employees.db`
- Generated 40 synthetic employee records
- Inserted records into the employees table

# Task 2: SQL Queries
Performed SQL queries to:
- Identify high-performing employees
- Filter employees based on salary range and department
- Calculate employee count and average salary by department

# Task 3: Pandas Analysis
- Loaded SQL data into Pandas DataFrame
- Calculated average performance score by department
- Replicated SQL query using Pandas filtering and sorting
- Compared SQL and Pandas for data analysis

# Key Insights
SQL is efficient for querying structured data stored in databases, while Pandas provides powerful tools for in-memory data analysis and transformation. Combining both tools allows flexible and efficient data analysis workflows.
